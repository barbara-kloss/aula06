Requisitos Não Funcionais

  • RNF01: O sistema deve garantir a segurança dos dados pessoais e financeiros dos usuários. (Essencial)
  • RNF02: O Sistema deve dar acesso contínuo 24/7 na rede. (Essencial)
  • RNF03: Tempo de resposta do sistema não deve exceder 2 segundos nas transações. (Importante)
  • RNF04: Interface do usuário intuitiva e responsiva, compatível para todos dispositivos. (Importante)
  • RNF05: O sistema deve ser capaz de suportar até 10.000 usuários simultâneos. (Importante)
  • RNF06: Compliance com regulamentos de proteção de dados, como GDPR. (Essencial)
  • RNF07: Implementação de backups diários dos dados. (Importante)
  • RNF08: O Sistema deve oferecer o suporte técnico 24/7 para usuários e hotéis parceiros. (Desejável)
  • RNF09: Atualizações do Sistema devem ocorrer sem interrupção significativa do serviço. (Desejável)
  • RNF10: Documentação completa e acessível para usuários e desenvolvedores. (Importante)
