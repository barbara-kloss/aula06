Prioridade dos Requisitos

Alta

  • O sistema deve permitir que usuários criem e gerenciem suas contas.
  • Usuários devem ser capazes de pesquisar hotéis por localização, datas de estadia, e número de hóspedes.
  • O sistema deve permitir a reserva de quartos com confirmação imediata.
  • Integração com sistemas de pagamento online para processar transações de reserva.
  • O sistema deve permitir cancelamento de reservas e processar reembolsos conforme a política de cancelamento do hotel.
  • O sistema deve garantir a segurança dos dados pessoais e financeiros dos usuários.
  • O Sistema deve dar acesso contínuo 24/7 na rede.
  • Compliance com regulamentos de proteção de dados, como GDPR.

Média

  • O sistema deve oferecer filtros de busca por preço, avaliação, e comodidades do hotel.
  • Usuários podem visualizar detalhes do hotel, incluindo fotos, avaliações e disponibilidade de quartos.
  • Usuários devem poder visualizar e modificar suas reservas atuais.
  • O sistema deve enviar e-mails de confirmação e lembretes das reservas.
  • Tempo de resposta do sistema não deve exceder 2 segundos nas transações.
  • Interface do usuário intuitiva e responsiva, compatível para todos dispositivos.
  • O sistema deve ser capaz de suportar até 10.000 usuários simultâneos.
  • Implementação de backups diários dos dados.
  • Documentação completa e acessível para usuários e desenvolvedores

Baixa

  • Suporte a múltiplos idiomas e moedas.
  • O Sistema deve oferecer o suporte técnico 24/7 para usuários e hotéis parceiros.
  • Atualizações do Sistema devem ocorrer sem interrupção significativa do serviço.
